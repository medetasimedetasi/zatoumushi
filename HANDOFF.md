## 0. これは何か(企画コンセプト)

薄緑・半透明のザトウムシ(harvestman / Opiliones)が、見下ろし1画面のステージ上で人間を捕まえて増えていく **見下ろし型の群れ育成シム**。

- 最初のプレイヤー個体(成体)が人間に触れると、その人間がザトウムシの赤ちゃん(子ども)になり群れが増える。
- ステージを左右に進めるほど難度が上がり、後半では人間が **銃で反撃** してくる。
- ビジュアルは「白い紙の上」のような明るい背景に、半透明の脚がぬるぬる動く、シンプルだが流麗な見た目を狙う。かなり遠目・見下ろしの定点カメラ。
- 発端はユーザー手描きのコンセプトアート(同フォルダの `ザトウムシ.png` / `ザトウムシシステム.png` / `.clip`)。

### ヘッドレス検証(重要)

Browserペインが非表示だと `requestAnimationFrame` が止まり、`computer{screenshot}` もタイムアウトする。回避のため index.html にデバッグフックを内蔵済み:

| フック | 用途 |
|---|---|
| `window.__step(n, dt=1/60)` | シミュレーションを n フレーム手動で進める(毎ステップ描画するので大きなnは遅い。~300が上限目安) |
| `window.__shot()` | 現在のcanvasを JPEG化して `/shot` にPOST → `D:\brender\Claude_code\shot.jpg` に保存。Readツールで画像確認できる |
| `window.__click(x, z)` | 成体の移動目標をワールド座標で指定 |
| `window.__stage(dir)` | `dir` +1/-1 でステージ遷移(`goToStage`直呼び) |
| `window.__zoom(v)` | 正射影フレームのスケール(0.3で寄り、1.5で引き)。検証時に個体を大写しにできる |
| `window.__state()` | 全状態をJSONで取得(stage/hp/gameOver/座標/弾数/remain/pounce/bodyY)。検証の主力 |
| `window.__damage(n)` | 成体にnダメージ(バランス調整・ゲームオーバー検証用) |

典型フロー: `navigate` でリロード → `__step(150)` → `__shot()` → Read `shot.jpg`。
(step は共有stateを進めるので、ライブ表示は状態がずれる点に留意)

## 2. 現在の仕様(実装済み)

### 2-1. カメラ / ワールド構造(第5更新で全面変更)
- **固定・正射影・ほぼ真上(仰角76°)・引き**。`frameCamera()`がアスペクト比からフレームを決める。
- **カメラは+z側から見下ろす**(第5更新で反転修正)。**画面右=+x=col+1、画面下=+z=row+1**。以前は左右が鏡像でステージ番号と移動方向が食い違うバグがあった。
- **4x4=16ステージのグリッドワールド**。開始`(0,0)`=ミニマップ左上、**敵本拠地`(3,3)`=右下(ミニマップで赤)**。`stageCol/stageRow`、キーは`skey(c,r)`。
- **難度 = マンハッタン距離 `stageDist(c,r)=c+r`**(0〜6)。
- **遷移は4方向**: 枠の**完全に外側**をクリックした時だけ`stageIntent={dx,dz}`が立ち、成体がその端に達すると`goToStage(dx,dz)`。枠内の人間・地面クリックでは絶対に遷移しない(旧: 端に張り付いた人間をクリックすると誤遷移するバグがあった)。グリッド外周は遷移不可で濃い線で塞ぐ(`sideLines`+`updateBlockedSides`)。行ける方向だけ`◀▲▼▶`矢印(テキストなし、`.off`/`.hot`)。
  - 成体は反対側の端から入場、**子どもは隊列を保って一緒に移動**。
  - 地形は`terrainOff=(col*4+row)*37.3`。岩は`mulberry32`のステージ固定シードで**再訪しても同じ配置**。
- **ミニマップ**(`#minimap`左下、4x4 divグリッド): 赤=本拠地、グレー=制圧済、緑枠=現在地。"STAGE n"テキスト表記は全廃。
- ホイールで`zoomScale`微調整のみ(基本は定点)。

### 2-2. ザトウムシの歩行(本物準拠・今回の目玉)
`class Harvestman`(L166)。脚は4対8本(`LEG_PAIRS` L152)。**論文・生態準拠に作ってある**:
- **第2脚(pair index 1、最長 len5.6)は歩行に使わず「触角」**。前方を左右に掃きながら上下に叩く(`sensory` 分岐、L更新内の sweep & tap)。盲人の白杖に例えられる実際の挙動。接地しない。
- **残り6本(pair 0,2,3)は交互三脚歩容(alternating tripod gait)**。`tripodGroup(side,pair)`(L159)で2グループに分け、`nextSwing` を交互に振り出す。閾値(foot が home から `0.42*scale` 超え)+ 速度で発火し、着地は速度方向へ予測(滑らない接地)。
- **2ボーンIK**: `placeSeg`(L131)で femur/tibia を配置。膝は「上+外」へ張り出す(`upOut`)ことでザトウムシ特有のアーチ。
- **胴の姿勢**: 歩脚の足高から平均Y・前後傾き・左右傾きを算出。走行時は正弦波状に上下(bob)。地形に対して体高と傾きが追従。
- マテリアル: 脚は半透明(opacity 0.7、触角脚は0.55でより淡い)、胴はやや濃色。

### 2-3. 捕食・群れ
- **成体・子どもの両方が捕食可能**(`step()` 内 capture ループ)。捕獲半径 `0.45*scale+0.28`。
- 人間を捕まえると `Harvestman(x,z,0.36,...)` の子どもを生成、群れに追加。
- **成体の移動はクリック指示が基本**。ただし**半径3.2以内の人間には自動で反応**する(ユーザー要望「触角の届く範囲だけ」)。それ以外は待機して触角だけ動かす。遠い獲物はクリック連打で追い込む操作感。
- **捕食はプレスモーション**: 触れると人間を`caught`にして押さえ込み、胴体を接地するまで下げる(台形カーブ、hMul最小0.15)。**成体は0.38秒で即取り込み(pounceT=0.5)、子どもは2.0〜2.6秒その場で捕食してから新しい子どもが生まれる(pounceT=0.85で取り込み)** — 実測: 成体0.17s/子ども2.08s。人間はもがくだけになり、押さえた個体は減速する。押さえ込み中の個体が撃たれたら人間は解放。ステージ遷移でも中断。
- **人間は有限POP**: `stageRemain{}`にステージ毎の残数を記録。狩り尽くすと「エリア制圧」表示、再POPなし。制圧済みステージに戻っても湧かない。新ステージ初訪問時のみ湧く。
- 子どもの目標: 近く(<8)の人間を追う、いなければ親の周りをうろつく。
- **分離(separation)実装済み**: 全個体ペアで反発(step内、minD=`0.62*(sa+sb)+0.12`)。成体は押されず子どもだけ避ける。settled時の最小間隔 ~0.88 を確認済み。
- 人間が全滅(かつ弾も無い)と新ウェーブを自動スポーン。

### 2-4. 人間・武器・戦闘(第5更新で大幅拡張)
`class Human(x, z, weapon, brave)`。
- 近く(<5)のザトウムシから逃げる。丸腰は全力で逃走、武装は距離を取りつつ撃つ。
- **武器3種**(`WEAPONS`、見た目で強さがわかる): **pistol**(赤茶、射程13)/ **rifle**(青灰、射程18・弾速17、長い銃身)/ **shotgun**(橙茶、射程7.5・3発扇状、太い銃身)。丸腰は黒。
- **難度テーブル**(`spawnHumans`/`pickWeapon`): 人数`5+2d`(本拠地16)、武装率`[0,.15,.35,.5,.65,.8]`(本拠地100%)、d3からrifle混入、d4からshotgun。スポーンは入場地点から7以上離す。
- **brave(守備隊)**: 本拠地の10人は要塞内に配置され**逃げない**。怯えなくても射程内なら撃つ。至近(<2)だけ少し離れる。
- **岩・要塞壁**(`buildRocks`): 弾を防ぐ遮蔽物。移動は`pushOutOfRocks`の押し出しで迂回(経路探索なし)。通常ステージは`3+d`個、本拠地は開口部2箇所の壁リング(半径6.4)+岩4。
- **子どもはHP2**(1発目は0.3秒無敵で耐える、2発目で死)。
- **制圧報酬: HP+2回復**(上限6)。**本拠地(3,3)制圧で GAME CLEAR**(クリックでリロード)。
- バランス実測: 子ども0で本拠地直行→16人中2人しか食えず約14秒で戦死(狙い通り)。序盤で群れを育て、制圧回復しながら進むのが勝ち筋。
- 弾 `spawnBullet`/`updateBullets`(L446/457): 直進、**子どもは即死、成体はHP制(6)**。
- **成体被弾**: HP-1・弾方向へノックバック(vel+3.5)・無敵0.9秒(マテリアル点滅)。HP0で `gameOver()` → "GAME OVER" 表示、クリックでリロード再開。HPはUI左上に♥♡で表示。
- 実プレイ知見: 子どもの群れが武装人間を先に食べてしまい「肉壁」として機能する(意図せぬ良ゲーム性なので活かす方向もあり)。

## 3. 既知の弱点・粗(次に直すと良い順)

~~1. 子どもの群れが密集しすぎる~~ → **分離実装済み**(第2セッション)
~~2. 成体が無敵~~ → **HP6・ノックバック・無敵時間・GAME OVER実装済み**(第2セッション)
~~4. 触角脚の揺動が機械的~~ → **個体差実装済み**(第2セッション)
~~7. シェーダー無し~~ → **リムライト実装済み**(第2セッション、強度はさらに調整余地あり)

1. **勝利/進行条件が曖昧**。有限POP+「エリア制圧」表示で改善したが、「全ステージ制圧でクリア」等のゴールはまだ無い。← 次の本命
2. **HP回復手段が無い**。捕食で回復 or ステージクリアで回復などの設計が要る(今は6発で終わり)。
3. **子どもが強すぎる**。武装人間を勝手に全部食べてしまい成体が撃たれる前に殲滅する。子どもは武装人間を怖がる/武装人間は子どもを優先して撃つ等のバランスが要検討。有限POPになったので「制圧済みステージでは子どもが餌不足になる」等の資源設計も絡められる。
4. カメラ仰角76°は「完全な真下」ではない。ユーザーの好み次第で90°化 or 微調整の余地(デフォルトズームは1.18に引き済み)。
5. パフォーマンスは軽い(サブ数ms/frame)が、捕食ループ・弾判定・分離が O(n²)。群れが数百規模になるなら空間分割(グリッド)が要る。
6. 音・UI・タイトル・セーブ等のゲーム外殻は一切なし(プロトタイプ段階)。

## 5. 知見

歩行の実装根拠(2026-07-11 調査):
- **交互三脚歩容(alternating tripod gait)**。片側の中脚+反対側の外脚を運び、残り3本で支える。走行時は速度・重心が正弦波状に振動。
- **第2脚が最も長く、歩行ではなく感覚器(触角)** として前方を掃き・叩いて探索する。
- 「長い脚・小さな胴」比率を模した六脚歩行ロボット **ASURA I** が、不整地踏破性向上を狙って開発され交互三脚・波状歩容を実証。

## 7. 引き継ぎ時の最初の一手(推奨)

1. `preview_start({name:"zatoumushi"})` で起動(ポート8124が他セッションのサーバーで使用中なら `preview_start({url:"http://localhost:8124"})` でそのまま接続できる)。`navigate` して `__step(200)`→`__shot()`→Read で現状把握。
2. 次の本命は **ステージクリア条件**(弱点1)と **HP回復設計**(弱点2)。
3. 変更は毎回 `__shot()` で視認確認、状態は `__state()` で数値確認(Browserペインが hidden でも可)。
4. `__step` は1回の呼び出しで合計300フレーム程度まで(それ以上はjavascript_toolが30秒でタイムアウトする)。長いシナリオは複数回に分割。JSは即時関数 `(()=>{...})()` で包むと `let` 再宣言エラーを避けられる。
